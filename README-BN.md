## 🚀 কেন এই প্রকল্প বিদ্যমান
এআই সিগন্যাল অত্যন্ত খণ্ডিত এবং দ্রুত চলে। ম্যানুয়াল ট্র্যাকিং সাধারণত এর কারণে ভেঙে যায়:

- খণ্ডিত চ্যানেল: কাগজপত্র, বিক্রেতা ঘোষণা, উপার্জন, নীতি আপডেট, এবং OSS প্রবণতা সংযোগ বিচ্ছিন্ন করা হয়েছে।
- সাম্প্রতিক গোলমাল: বাসি খবর পুনরুত্থিত হতে থাকে এবং নতুন সংকেত দেয়।
- ডিডুপ্লিকেশন ব্যথা: সিন্ডিকেটেড রিপোস্টগুলি বারবার ইনজেশন এবং বারবার বিজ্ঞপ্তি ট্রিগার করে।
- অটোমেশন গ্যাপ: এজেন্ট পাইপলাইনে সরাসরি "বুদ্ধিমত্তা আনা" সংহত করা কঠিন।

`OpenHawk` এটিকে একটি সর্বদা চালু, এক্সটেনসিবল ইন্টেলিজেন্স লেয়ারে পরিণত করে যা এজেন্টরা সরাসরি কল করতে পারে।

## 👥 এটা কার জন্য
- এআই গবেষকরা: ক্রমাগত কাগজপত্র এবং পদ্ধতির বিবর্তন ট্র্যাক করুন।
- পণ্য এবং প্রকৌশল দল: ট্র্যাক রিলিজ, টুলিং, এবং বিকাশকারী সংকেত।
- বিনিয়োগ এবং কৌশল দল: ট্র্যাক উপার্জন, ক্যাপেক্স, অর্থায়ন, এবং বাজারের গতিবিধি।
- পলিসি এবং কমপ্লায়েন্স টিম: ট্র্যাক রেগুলেশন এবং এআই সুরক্ষা ঘটনাগুলি অঞ্চল জুড়ে৷
- এজেন্ট নির্মাতা: প্রোগ্রামযোগ্য ক্ষমতা হিসাবে MCP/CLI টুল ব্যবহার করুন।

## ⚙️ মূল ক্ষমতা
- অঞ্চল-ভিত্তিক উত্স সংস্থার সাথে মাল্টি-সোর্স ইনজেশন।
- পুনরাবৃত্ত আনা/ধাক্কা রোধ করতে অবিরাম ইতিহাস + ডিডআপ সূচক।
- বাসি আইটেম দমন করার জন্য রিসেন্সি নিয়ন্ত্রণ (উদাহরণস্বরূপ, একটি 90-দিনের উইন্ডো)।
- শিরোনাম, এলএলএম বিশ্লেষণ এবং পুশ বিষয়বস্তুর জন্য ইউনিফাইড অনুবাদ পাইপলাইন (যে কোনো লক্ষ্য ভাষা)।
- মাল্টি-চ্যানেল বিজ্ঞপ্তি: `feishu`, `wework`, `wechat`, `telegram`, `dingtalk`, `ntfy`, `bark`, `slack`, `email`।
- স্মার্ট পুশ কৌশল: `দৈনিক` / `বৃদ্ধিমূলক` / `রিয়েলটাইম`।
- নির্ধারিত আনয়ন এবং ঐচ্ছিক অটো-পুশ সাবস্ক্রিপশন কাজ।
- প্রোটোকল-ভিত্তিক এজেন্ট ইন্টিগ্রেশনের জন্য MCP টুল ইন্টারফেস।
- এজেন্ট CLI (নতুন) সরাসরি, স্ক্রিপ্টেবল স্থানীয় টুল আহ্বানের জন্য।

## 🌐 ইউনিফাইড ট্রান্সলেশন পাইপলাইন (যেকোনো টার্গেট ভাষা)
- সমস্ত স্তর জুড়ে একটি পাইপলাইন: `শিরোনাম`, `এলএলএম সারাংশ/বিশ্লেষণ`, এবং `বিজ্ঞপ্তি পেলোড`।
- টার্গেট ভাষা সমর্থন: `ইংরেজি`, `কোরিয়ান`, `জাপানি`, `ফরাসি`, `চীনা`, `ঐতিহ্যবাহী চীনা`, প্লাস কাস্টম ভাষা লক্ষ্য।
- খরচ নিয়ন্ত্রণ: ব্যাচ অনুবাদ, ক্রমবর্ধমান ক্ষেত্র পূরণ, এবং পুনরাবৃত্তি API কল এড়াতে অবিরত পুনঃব্যবহার।
- সামঞ্জস্যপূর্ণ আচরণ: ওয়েব, MCP এবং CLI একই `আউটপুট_ভাষা` শব্দার্থ ভাগ করে, তাই UI এবং পুশ আউটপুট সিঙ্কে থাকে।

উদাহরণ (CLI):
```bash
# Set AI Finance page output language to Japanese
openhawk-ai-cli call save_scope_settings --args '{"scope":"market_finance","output_language":"Japanese"}'

# Fetch using this scope and language policy
openhawk-ai-cli call fetch_scope_items --args '{"scope":"market_finance","max_per_source":20}'
```

## ⚡ দ্রুত শুরু
### 1) 🧩 প্রয়োজনীয়তা
- পাইথন `>= 3.12`
- `uv` সুপারিশ করা হয়
- ডকার ব্যবহারকারী: `ডকার` + `ডকার কম্পোজ`

### 2) 🖥️ স্থানীয় স্টার্টআপ (উন্নয়ন)
```bash
uv sync --locked
```

1. মেইন ফেচ/রানটাইম একবার চালান:
```bash
openhawk-ai
```

2. ইন্টারেক্টিভ ওয়েব কনসোল চালান (6-পৃষ্ঠা UI):
```bash
python -m openhawk_ai.web.panel_server --port 8080 --output-dir output
```

3. MCP পরিষেবা শুরু করুন (HTTP):
```bash
openhawk-ai-mcp --transport http --host 0.0.0.0 --port 3333
```

### 3) 🐳 ডকার স্টার্ট
```bash
docker compose -f docker/docker-compose.yml up -d --build
```

ডিফল্ট পোর্ট:
- ওয়েব পরিষেবা: `WEBSERVER_PORT` (ডিফল্ট: `8080`)
- MCP শেষ পয়েন্ট: `http://127.0.0.1:3333/mcp`

থামুন:
```bash
docker compose -f docker/docker-compose.yml down
```

লগ দেখুন:
```bash
docker compose -f docker/docker-compose.yml logs -f
```

## 🤖 এজেন্ট CLI (নতুন)
এজেন্ট/স্ক্রিপ্টগুলিকে MCP পরিবহন ছাড়াই সরাসরি OpenHawk টুলগুলিকে কল করতে দিতে, এই রেপোটি `openhawk-ai-cli` যোগ করে।

### 🎯 ডিজাইন লক্ষ্য
- এমসিপি সরঞ্জামগুলির মতো একই ক্ষমতার পৃষ্ঠ (একই সরঞ্জামের নাম এবং আর্গুমেন্ট শব্দার্থবিদ্যা)।
- অটোমেশন-বান্ধব ইন্টিগ্রেশনের জন্য JSON ইনপুট এবং JSON আউটপুট।
- শেল স্ক্রিপ্ট, CI পাইপলাইন এবং এজেন্ট নির্বাহকদের সাথে ভাল কাজ করে।

### 🧪 বেসিক কমান্ড
```bash
openhawk-ai-cli tools
```

### 📌 সাধারণ উদাহরণ
1. সমস্ত উপলব্ধ সরঞ্জাম এবং প্যারামের তালিকা করুন:
```bash
openhawk-ai-cli tools
```

2. ইনলাইন JSON সহ একটি টুল কল করুন:
```bash
openhawk-ai-cli call list_scope_items --args '{"scope":"market_finance","limit":20}'
```

3. args ফাইল সহ একটি টুল কল করুন:
```bash
openhawk-ai-cli call upsert_scope_subscription --args-file ./payload.json
```

4. প্রকল্প রুট এবং আউটপুট ডিরেক্টরি ওভাররাইড করুন:
```bash
openhawk-ai-cli --project-root . --output-dir ./output call get_project_overview
```

5. কমপ্যাক্ট JSON আউটপুট (পাইপলাইন-বান্ধব):
```bash
openhawk-ai-cli call list_scopes --compact
```

### উইন্ডোজ পাওয়ারশেল উদাহরণ (প্রস্তাবিত)
1. `--args-file` এর সাথে `ConvertTo-Json` ব্যবহার করুন (সবচেয়ে নির্ভরযোগ্য):
```powershell
$payload = @{ scope = "market_finance"; limit = 20 } | ConvertTo-Json -Compress
$payload | Set-Content -Encoding utf8 .\payload.json
openhawk-ai-cli call list_scope_items --args-file .\payload.json --compact
```

2. একটি এখানে-স্ট্রিং args ফাইল ব্যবহার করুন:
```powershell
@'
{
  "scope": "frontier",
  "max_per_source": 20,
  "source_ids": ["openai-news", "anthropic-news"]
}
'@ | Set-Content -Encoding utf8 .\payload.json

openhawk-ai-cli call fetch_scope_items --args-file .\payload.json --compact
```

3. যে সরঞ্জামগুলির কোন আর্গ প্রয়োজন নেই সরাসরি কল করা যাবে:
```powershell
openhawk-ai-cli call get_project_overview --compact
```

### 🧾 প্রস্থান কোড
- `0`: সাফল্য
- `1`: রানটাইম/টুল এক্সিকিউশন ত্রুটি
- `2`: অবৈধ প্যারাম, অজানা টুল, বা বিকৃত JSON

### 🛠️ CLI প্যারামিটার এবং কভারেজ
CLI স্থির আর্গুমেন্ট:

| স্তর | প্যারামিটার | বর্ণনা |
| --- | --- | --- |
| গ্লোবাল | `--প্রকল্প-মূল` | প্রকল্প রুট ওভাররাইড করুন |
| গ্লোবাল | `--আউটপুট-ডির` | আউটপুট ডিরেক্টরি ওভাররাইড করুন |
| গ্লোবাল | `---কম্প্যাক্ট` | কমপ্যাক্ট JSON নির্গত করুন |
| সাবকমান্ড | 'সরঞ্জাম' | উপলব্ধ সরঞ্জাম তালিকা |
| সাবকমান্ড | `কল <টুল>` | একটি টুল আহ্বান করুন |
| 'কল' বিকল্প | `--আর্গস` | ইনলাইন JSON args |
| 'কল' বিকল্প | `--args-file` | ফাইল থেকে JSON args লোড করুন |

টুল ব্যবসায়িক আর্গুমেন্ট টুল প্রতি সংজ্ঞায়িত করা হয়. ব্যবহার করুন:

```bash
openhawk-ai-cli tools --compact
```

কভারেজ সীমানা:
- ওভারভিউ, স্কোপ আনয়ন/তালিকা/সেটিংস/সাবস্ক্রিপশন, এবং পেপার ইন্টেলিজেন্স/সাবস্ক্রিপশন সহ বর্তমানে প্রকাশিত সমস্ত MCP টুল ক্ষমতা (22 টুল) কভার করে।
- প্রসেস/কন্টেইনার লাইফসাইকেল সরাসরি পরিচালনা করে না (উদাহরণস্বরূপ ডকার বা ওয়েব সার্ভার প্রক্রিয়া শুরু/বন্ধ করা)।
- ব্রাউজার UI মিথস্ক্রিয়া সরাসরি সঞ্চালন করে না, তবে সমতুল্য ডেটা-লেয়ার অপারেশনগুলি (সেটিংস, আনয়ন, সদস্যতা, পুশ) কভার করে।

## ❓ প্রায়শই জিজ্ঞাসিত প্রশ্ন
### প্রশ্ন 1: CLI এবং MCP এর মধ্যে সম্পর্ক কি?
CLI একই MCP-সামঞ্জস্যপূর্ণ টুল ফাংশন সরাসরি কল করে। প্রোটোকল ইন্টিগ্রেশনের জন্য MCP এবং স্থানীয় স্ক্রিপ্টিং/অটোমেশনের জন্য CLI ব্যবহার করুন।

### প্রশ্ন 2: CLI সব বৈশিষ্ট্য কভার করতে পারে?
CLI বর্তমানে MCP (22 টুল) দ্বারা উন্মুক্ত সমস্ত ক্ষমতা কভার করে। যদি UI ম্যাপে কোনো অ্যাকশন ডেটা-লেয়ার অপারেশনে (আনয়ন, ক্যোয়ারী, সেটিংস, সাবস্ক্রিপশন, পুশ) হয়, তাহলে এটি সাধারণত CLI-এর মাধ্যমে স্বয়ংক্রিয় হয়।

### প্রশ্ন 3: আমি কেন ডুপ্লিকেট মেসেজ দেখছি?
চেক করুন:
- `output/ai_progress_seen.json` সঠিকভাবে স্থায়ী/মাউন্ট করা আছে কিনা
- ডুপ্লিকেট সাবস্ক্রিপশন নিয়ম কনফিগার করা হয়েছে কিনা
- `টাইমলাইন` এবং পৃষ্ঠা-স্তরের সদস্যতা ওভারল্যাপিং উইন্ডোগুলিকে ট্রিগার করছে কিনা

### প্রশ্ন 4: ন্যূনতম এজেন্ট ইন্টিগ্রেশন পাথ কি?
দিয়ে শুরু করুন:
```bash
openhawk-ai-cli call get_project_overview
```
তারপর প্রয়োজন অনুযায়ী `লিস্ট_স্কোপ_আইটেম` / `লিস্ট_পেপার` / `রান__*_সাবস্ক্রিপশন` কল করুন।
