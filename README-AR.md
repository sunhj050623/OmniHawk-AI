## 🚀 لماذا يوجد هذا المشروع
إشارات الذكاء الاصطناعي مجزأة للغاية وتتحرك بسرعة. عادةً ما يتعطل التتبع اليدوي للأسباب التالية:

- القنوات المجزأة: يتم فصل الأوراق وإعلانات البائعين والأرباح وتحديثات السياسة واتجاهات OSS.
- ضجيج الحداثة: تستمر الأخبار القديمة في الظهور وتزاحم الإشارة الجديدة.
- ألم إلغاء البيانات المكررة: تؤدي عمليات إعادة النشر المشتركة إلى تكرار العرض والإشعارات المتكررة.
- فجوة الأتمتة: من الصعب دمج "جلب المعلومات الاستخبارية" مباشرة في مسارات العملاء.

يقوم OmniHawk AI بتحويل هذا إلى طبقة ذكاء قابلة للتوسيع ودائمة التشغيل يمكن للعملاء الاتصال بها مباشرة.

## 👥 لمن
- باحثو الذكاء الاصطناعي: يتابعون الأبحاث وتطور الأساليب بشكل مستمر.
- فرق المنتج والهندسة: تتبع الإصدارات والأدوات وإشارات المطورين.
- فرق الاستثمار والاستراتيجية: تتبع الأرباح والنفقات الرأسمالية والتمويل وحركات السوق.
- فرق السياسة والامتثال: تتبع حوادث التنظيم وسلامة الذكاء الاصطناعي عبر المناطق.
- منشئو الوكلاء: استخدم أدوات MCP/CLI كقدرات قابلة للبرمجة.

## ⚙️ القدرات الأساسية
- استيعاب متعدد المصادر مع تنظيم المصدر الموجه نحو المنطقة.
- السجل المستمر + فهارس dedup لمنع تكرار عمليات الجلب/الدفع.
- التحكم في الحداثة (على سبيل المثال، نافذة مدتها 90 يومًا) لمنع العناصر التي لا معنى لها.
- خط أنابيب الترجمة الموحد (أي لغة مستهدفة) للعناوين وتحليل LLM والمحتوى المدفوع.
- إشعارات متعددة القنوات: `feishu`، `wework`، `wechat`، `telegram`، `dingtalk`، `ntfy`، `bark`، `slack`، `email`.
- استراتيجيات الدفع الذكية: "يوميًا" / "تزايديًا" / "في الوقت الفعلي".
- الجلب المجدول ومهام الاشتراك التلقائي الاختيارية.
- واجهة أداة MCP لتكامل الوكيل القائم على البروتوكول.
- وكيل CLI (جديد) لاستدعاء أداة محلية مباشرة وقابلة للبرمجة.

## 🌐 خط الترجمة الموحد (أي لغة مستهدفة)
- مسار واحد عبر جميع الطبقات: "العنوان"، و"ملخص/تحليل LLM"، و"حمولة الإشعارات".
- دعم اللغة المستهدفة: `الإنجليزية`، `الكورية`، `اليابانية`، `الفرنسية`، `الصينية`، `الصينية التقليدية`، بالإضافة إلى أهداف اللغة المخصصة.
- التحكم في التكلفة: ترجمة الدفعات، وملء الحقول المتزايد، وإعادة الاستخدام المستمر لتجنب استدعاءات واجهة برمجة التطبيقات المتكررة.
- السلوك المتسق: يشترك الويب وMCP وCLI في نفس دلالات "لغة_الإخراج"، بحيث تظل واجهة المستخدم ومخرجات الدفع متزامنة.

مثال (CLI):
```bash
# Set AI Finance page output language to Japanese
omnihawk-ai-cli call save_scope_settings --args '{"scope":"market_finance","output_language":"Japanese"}'

# Fetch using this scope and language policy
omnihawk-ai-cli call fetch_scope_items --args '{"scope":"market_finance","max_per_source":20}'
```

## ⚡ بداية سريعة
### 1) 🧩 المتطلبات
- بايثون `>= 3.12`
- يوصى بـ `uv`
- مستخدمو Docker: `Docker` + `Docker Compose`

### 2) 🖥️ الشركات الناشئة المحلية (التطوير)
```bash
uv sync --locked
```

1. قم بتشغيل عملية الجلب/وقت التشغيل الرئيسية مرة واحدة:
```bash
omnihawk-ai
```

2. قم بتشغيل وحدة تحكم الويب التفاعلية (واجهة مستخدم مكونة من 6 صفحات):
```bash
python -m omnihawk_ai.web.panel_server --port 8080 --output-dir output
```

3. ابدأ خدمة MCP (HTTP):
```bash
omnihawk-ai-mcp --transport http --host 0.0.0.0 --port 3333
```

### 3) 🐳 بداية عامل الميناء
```bash
docker compose -f docker/docker-compose.yml up -d --build
```

المنافذ الافتراضية:
- خدمة الويب: `WEBSERVER_PORT` (الافتراضي: `8080`)
- نقطة نهاية MCP: `http://127.0.0.1:3333/mcp`

قف:
```bash
docker compose -f docker/docker-compose.yml down
```

عرض السجلات:
```bash
docker compose -f docker/docker-compose.yml logs -f
```

## 🤖 الوكيل CLI (جديد)
للسماح للوكلاء/البرامج النصية باستدعاء أدوات OmniHawk مباشرةً دون نقل MCP، يضيف هذا الريبو `omnihawk-ai-cli`.

### 🎯 أهداف التصميم
- نفس سطح القدرة مثل أدوات MCP (نفس أسماء الأدوات ودلالات الوسيطات).
- إدخال JSON وإخراج JSON للتكامل السهل مع الأتمتة.
- يعمل بشكل جيد مع البرامج النصية لـ Shell وخطوط أنابيب CI ومنفذي الوكلاء.

### 🧪 الأمر الأساسي
```bash
omnihawk-ai-cli tools
```

### 📌أمثلة شائعة
1. قم بإدراج جميع الأدوات والمعلمات المتاحة:
```bash
omnihawk-ai-cli tools
```

2. اتصل بأداة تحتوي على JSON المضمّن:
```bash
omnihawk-ai-cli call list_scope_items --args '{"scope":"market_finance","limit":20}'
```

3. استدعاء أداة باستخدام ملف args:
```bash
omnihawk-ai-cli call upsert_scope_subscription --args-file ./payload.json
```

4. تجاوز جذر المشروع ودليل الإخراج:
```bash
omnihawk-ai-cli --project-root . --output-dir ./output call get_project_overview
```

5. مخرجات JSON مدمجة (صديقة لخطوط الأنابيب):
```bash
omnihawk-ai-cli call list_scopes --compact
```

### أمثلة على Windows PowerShell (مستحسن)
1. استخدم "ConvertTo-Json" مع "--args-file" (الأكثر موثوقية):
```powershell
$payload = @{ scope = "market_finance"; limit = 20 } | ConvertTo-Json -Compress
$payload | Set-Content -Encoding utf8 .\payload.json
omnihawk-ai-cli call list_scope_items --args-file .\payload.json --compact
```

2. استخدم ملف وسيطات السلسلة هنا:
```powershell
@'
{
  "scope": "frontier",
  "max_per_source": 20,
  "source_ids": ["openai-news", "anthropic-news"]
}
'@ | Set-Content -Encoding utf8 .\payload.json

omnihawk-ai-cli call fetch_scope_items --args-file .\payload.json --compact
```

3. يمكن استدعاء الأدوات التي لا تحتاج إلى وسيطات مباشرةً:
```powershell
omnihawk-ai-cli call get_project_overview --compact
```

### 🧾 رموز الخروج
- `0`: النجاح
- `1`: خطأ في وقت التشغيل/تنفيذ الأداة
- `2`: معلمات غير صالحة، أو أداة غير معروفة، أو تنسيق JSON غير صحيح

### 🛠️ معلمات CLI والتغطية
وسيطات CLI الثابتة:

| مستوى | المعلمة | وصف |
| --- | --- | --- |
| عالمي | `--جذر المشروع` | تجاوز جذر المشروع |
| عالمي | `--إخراج دير` | تجاوز دليل الإخراج |
| عالمي | `--مضغوط` | تنبعث منها JSON المدمجة |
| أمر فرعي | "الأدوات". | قائمة الأدوات المتاحة |
| أمر فرعي | `استدعاء <أداة>` | استدعاء أداة |

يتم تعريف وسيطات أعمال الأداة لكل أداة. يستخدم:

```bash
omnihawk-ai-cli tools --compact
```

حدود التغطية:
- يغطي جميع إمكانيات أداة MCP المكشوفة حاليًا (22 أداة) بما في ذلك النظرة العامة وجلب النطاق/القائمة/الإعدادات/الاشتراكات والاستخبارات الورقية/الاشتراكات.

## ❓الأسئلة الشائعة
### س1: ما هي العلاقة بين CLI وMCP؟
تستدعي واجهة سطر الأوامر (CLI) نفس وظائف الأداة المتوافقة مع MCP مباشرةً. استخدم MCP لتكامل البروتوكول وCLI للبرمجة النصية/الأتمتة المحلية.

### السؤال الثاني: هل يمكن لواجهة سطر الأوامر (CLI) تغطية جميع الميزات؟
تغطي واجهة سطر الأوامر (CLI) جميع الإمكانات المكشوفة حاليًا بواسطة MCP (22 أداة). إذا كان هناك إجراء في واجهة المستخدم يتم تعيينه لعمليات طبقة البيانات (الجلب، الاستعلام، الإعدادات، الاشتراكات، الدفع)، فإنه عادةً ما يكون قابلاً للتشغيل الآلي عبر سطر الأوامر.

### س3: لماذا أرى رسائل مكررة؟
يفحص:
- ما إذا كان `output/ai_progress_seen.json` مستمرًا/مثبتًا بشكل صحيح
