## 🚀 이 프로젝트가 존재하는 이유
AI 신호는 고도로 단편화되어 있으며 빠르게 움직입니다. 수동 추적은 일반적으로 다음과 같은 이유로 인해 중단됩니다.

- 단편화된 채널: 논문, 공급업체 발표, 수입, 정책 업데이트 및 OSS 동향이 단절되어 있습니다.
- 최신 노이즈: 진부한 뉴스가 계속해서 다시 떠오르고 새로운 신호를 몰아냅니다.
- 중복 제거 문제: ​​신디케이트된 재게시로 인해 반복적인 수집 및 알림이 발생합니다.
- 자동화 격차: "지능 가져오기"를 에이전트 파이프라인에 직접 통합하기가 어렵습니다.

'OmniHawk AI'는 이를 에이전트가 직접 호출할 수 있는 상시 확장 가능한 인텔리전스 계층으로 전환합니다.

## 👥 누구를 위한 것인가
- AI 연구자: 논문과 방법의 발전을 지속적으로 추적합니다.
- 제품 및 엔지니어링 팀: 릴리스, 도구 및 개발자 신호를 추적합니다.
- 투자 및 전략 팀: 수익, 자본 지출, 자금 조달 및 시장 동향을 추적합니다.
- 정책 및 규정 준수 팀: 지역 전반에 걸쳐 규제 및 AI 안전 사고를 추적합니다.
- 에이전트 빌더: MCP/CLI 도구를 프로그래밍 가능한 기능으로 사용합니다.

## 🧭 6개의 독립 페이지(병렬, 중첩되지 않음)
| 페이지 | 주요 용도 | 일반적인 소스 |
| --- | --- | --- |
| AI 종이 레이더 | 학업 추적 및 심층 논문 분석 | arXiv 및 연구 피드 |
| AI 프론티어 | 모델, 제품 및 기술 업데이트 | 공식 사이트, 공급업체 블로그, 연구소 |
| AI 금융 | 자본 시장 및 회사 인텔리전스 | 수익, 통화 기록, 시장 피드, 파이낸싱/M&A |
| AI 산업 보고서 | 산업 및 기관 연구 | 글로벌 싱크탱크, 기관, 백서 |
| AI 정책 및 안전 | 규제 및 위험 모니터링 | 규제 기관, 정책 기관, 안전 사고 소스 |
| AI OSS 및 개발 신호 | 오픈 소스 도구 및 개발 생태계 동향 | GitHub 트렌드 + README 의미 체계 필터링 |

> 6개 페이지는 모두 독립적입니다. 각 페이지에는 격리된 설정, 구독 및 푸시 규칙이 있습니다.

## ⚙️ 핵심 기능
- 지역 중심의 소스 구성으로 멀티 소스 수집.
- 반복적인 가져오기/푸시를 방지하기 위한 지속적인 기록 + 중복 제거 인덱스.
- 오래된 항목을 억제하는 최신성 제어(예: 90일 기간).
- 제목, LLM 분석 및 푸시 콘텐츠에 대한 통합 번역 파이프라인(모든 대상 언어).
- 다중 채널 알림: `feishu`, `wework`, `wechat`, `telegram`, `dingtalk`, `ntfy`, `bark`, `slack`, `email`.
- 스마트 푸시 전략: `매일` / `증분` / `실시간`.
- 예약된 가져오기 및 선택적 자동 푸시 구독 작업.
- 프로토콜 기반 에이전트 통합을 위한 MCP 도구 인터페이스입니다.
- 스크립트 가능한 직접 로컬 도구 호출을 위한 에이전트 CLI(신규)

## 🌐 통합 번역 파이프라인(모든 대상 언어)
- 모든 레이어에 걸쳐 하나의 파이프라인: '제목', 'LLM 요약/분석', '알림 페이로드'.
- 대상 언어 지원: `영어`, `한국어`, `일본어`, `프랑스어`, `중국어`, `중국어 번체` 및 사용자 정의 언어 대상.
- 비용 관리: 일괄 번역, 증분 필드 채우기 및 지속적인 재사용을 통해 반복적인 API 호출을 방지합니다.
- 일관된 동작: 웹, MCP 및 CLI는 동일한 'output_언어' 의미를 공유하므로 UI ​​및 푸시 출력이 동기화된 상태로 유지됩니다.

예(CLI):
```bash
# Set AI Finance page output language to Japanese
omnihawk-ai-cli call save_scope_settings --args '{"scope":"market_finance","output_language":"Japanese"}'

# Fetch using this scope and language policy
omnihawk-ai-cli call fetch_scope_items --args '{"scope":"market_finance","max_per_source":20}'
```

## ⚡ 빠른 시작
### 1) 🧩 요구사항
- 파이썬 `>= 3.12`
- `uv`를 권장합니다
- Docker 사용자: `Docker` + `Docker Compose`

### 2) 🖥️ 로컬 스타트업(개발)
```bash
uv sync --locked
```

1. 기본 가져오기/런타임을 한 번 실행합니다.
```bash
omnihawk-ai
```

2. 대화형 웹 콘솔(6페이지 UI)을 실행합니다.
```bash
python -m omnihawk_ai.web.panel_server --port 8080 --output-dir output
```

3. MCP 서비스(HTTP)를 시작합니다.
```bash
omnihawk-ai-mcp --transport http --host 0.0.0.0 --port 3333
```

### 3) 🐳 도커 시작
```bash
docker compose -f docker/docker-compose.yml up -d --build
```

기본 포트:
- 웹 서비스: `WEBSERVER_PORT` (기본값: `8080`)
- MCP 엔드포인트: `http://127.0.0.1:3333/mcp`

멈추다:
```bash
docker compose -f docker/docker-compose.yml down
```

로그 보기:
```bash
docker compose -f docker/docker-compose.yml logs -f
```

## 🤖 에이전트 CLI(신규)
에이전트/스크립트가 MCP 전송 없이 OmniHawk 도구를 직접 호출할 수 있도록 이 저장소에는 'omnihawk-ai-cli'가 추가됩니다.

### 🎯 디자인 목표
- MCP 도구와 동일한 기능 표면(동일한 도구 이름 및 인수 의미)
- 자동화 친화적인 통합을 위한 JSON 입력 및 JSON 출력.
- 셸 스크립트, CI 파이프라인 및 에이전트 실행기와 잘 작동합니다.

### 🧪 기본 명령
```bash
omnihawk-ai-cli tools
```

### 😀 일반적인 예
1. 사용 가능한 모든 도구 및 매개변수를 나열합니다.
```bash
omnihawk-ai-cli tools
```

2. 인라인 JSON을 사용하여 도구를 호출합니다.
```bash
omnihawk-ai-cli call list_scope_items --args '{"scope":"market_finance","limit":20}'
```

3. args 파일을 사용하여 도구를 호출합니다.
```bash
omnihawk-ai-cli call upsert_scope_subscription --args-file ./payload.json
```

4. 프로젝트 루트 및 출력 디렉터리를 재정의합니다.
```bash
omnihawk-ai-cli --project-root . --output-dir ./output call get_project_overview
```

5. 컴팩트 JSON 출력(파이프라인 친화적):
```bash
omnihawk-ai-cli call list_scopes --compact
```

### Windows PowerShell 예제(권장)
1. `--args-file`과 함께 `ConvertTo-Json`을 사용합니다(가장 안정적임).
```powershell
$payload = @{ scope = "market_finance"; limit = 20 } | ConvertTo-Json -Compress
$payload | Set-Content -Encoding utf8 .\payload.json
omnihawk-ai-cli call list_scope_items --args-file .\payload.json --compact
```

2. 여기에 문자열 args 파일을 사용합니다.
```powershell
@'
{
  "scope": "frontier",
  "max_per_source": 20,
  "source_ids": ["openai-news", "anthropic-news"]
}
'@ | Set-Content -Encoding utf8 .\payload.json

omnihawk-ai-cli call fetch_scope_items --args-file .\payload.json --compact
```

3. 인수가 필요 없는 도구는 직접 호출할 수 있습니다.
```powershell
omnihawk-ai-cli call get_project_overview --compact
```

### 🧾 종료 코드
- `0`: 성공
- `1`: 런타임/도구 실행 오류
- `2`: 잘못된 매개변수, 알 수 없는 도구 또는 잘못된 JSON

### 🛠️ CLI 매개변수 및 적용 범위
CLI 고정 인수:

| 수준 | 매개변수 | 설명 |
| --- | --- | --- |
| 글로벌 | `--프로젝트-루트` | 프로젝트 루트 재정의 |
| 글로벌 | `--출력-디렉터리` | 출력 디렉터리 재정의 |
| 글로벌 | `--컴팩트` | 컴팩트 JSON 내보내기 |
| 하위 명령 | '도구' | 사용 가능한 도구 나열 |
| 하위 명령 | `<도구> 호출` | 도구 호출 |
| '통화' 옵션 | `--args` | 인라인 JSON 인수 |
| '통화' 옵션 | `--args-파일` | 파일에서 JSON 인수 로드 |

도구 비즈니스 인수는 도구별로 정의됩니다. 사용:

```bash
omnihawk-ai-cli tools --compact
```

적용 범위:
- 개요, 범위 가져오기/목록/설정/구독 및 문서 인텔리전스/구독을 포함하여 현재 노출된 모든 MCP 도구 기능(22개 도구)을 다룹니다.
- 프로세스/컨테이너 수명주기(예: Docker 또는 웹 서버 프로세스 시작/중지)를 직접 관리하지 않습니다.
- 브라우저 UI 상호 작용을 직접 수행하지 않지만 동등한 데이터 계층 작업(설정, 가져오기, 구독, 푸시)을 다룹니다.

## ❓FAQ
### Q1: CLI와 MCP는 어떤 관계인가요?
CLI는 동일한 MCP 호환 도구 기능을 직접 호출합니다. 프로토콜 통합에는 MCP를 사용하고 로컬 스크립팅/자동화에는 CLI를 사용합니다.

### Q2: CLI가 모든 기능을 다룰 수 있나요?
CLI는 현재 MCP(22개 도구)에서 제공하는 모든 기능을 다룹니다. UI의 작업이 데이터 계층 작업(가져오기, 쿼리, 설정, 구독, 푸시)에 매핑되는 경우 일반적으로 CLI를 통해 자동화할 수 있습니다.

### Q3: 메시지가 중복되는 이유는 무엇입니까?
확인하다:
- `output/ai_progress_seen.json`이 올바르게 지속/마운트되었는지 여부
- 중복 구독 규칙이 구성되어 있는지 여부
- '타임라인' 및 페이지 수준 구독이 겹치는 창을 트리거하는지 여부

### Q4: 최소 에이전트 통합 경로는 무엇인가요?
다음으로 시작하세요:
```bash
omnihawk-ai-cli call get_project_overview
```
그런 다음 필요에 따라 `list_scope_items` / `list_papers` / `run_*_subscriptions`를 호출하세요.
