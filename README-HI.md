## 🚀 यह प्रोजेक्ट क्यों अस्तित्व में है
एआई सिग्नल अत्यधिक खंडित होते हैं और तेजी से चलते हैं। मैन्युअल ट्रैकिंग आमतौर पर निम्न कारणों से विफल हो जाती है:

- खंडित चैनल: कागजात, विक्रेता घोषणाएं, कमाई, नीति अपडेट और ओएसएस रुझान डिस्कनेक्ट हो गए हैं।
- ताज़ा शोर: पुरानी ख़बरें बार-बार सामने आती रहती हैं और नए संकेत देती रहती हैं।
- डिडुप्लीकेशन दर्द: सिंडिकेटेड रिपोस्ट बार-बार अंतर्ग्रहण और बार-बार नोटिफिकेशन को ट्रिगर करता है।
- स्वचालन अंतर: "खुफिया जानकारी" को सीधे एजेंट पाइपलाइनों में एकीकृत करना मुश्किल है।

'ओम्निहॉक एआई' इसे हमेशा चालू रहने वाली, एक्स्टेंसिबल इंटेलिजेंस परत में बदल देता है जिसे एजेंट सीधे कॉल कर सकते हैं।

## 👥 यह किसके लिए है
- एआई शोधकर्ता: कागजात और विधि विकास को लगातार ट्रैक करें।
- उत्पाद और इंजीनियरिंग टीमें: ट्रैक रिलीज़, टूलींग और डेवलपर सिग्नल।
- निवेश और रणनीति टीमें: कमाई, पूंजीगत व्यय, वित्तपोषण और बाजार की गतिविधियों पर नज़र रखें।
- नीति और अनुपालन टीमें: विभिन्न क्षेत्रों में विनियमन और एआई सुरक्षा घटनाओं पर नज़र रखें।
- एजेंट बिल्डर्स: प्रोग्रामयोग्य क्षमताओं के रूप में एमसीपी/सीएलआई टूल का उपयोग करें।

## ⚙️ मुख्य क्षमताएँ
- क्षेत्र-उन्मुख स्रोत संगठन के साथ बहु-स्रोत अंतर्ग्रहण।
- बार-बार फ़ेच/पुश को रोकने के लिए लगातार इतिहास + डिडअप इंडेक्स।
- पुरानी वस्तुओं को दबाने के लिए रीसेंसी नियंत्रण (उदाहरण के लिए, 90-दिन की विंडो)।
- शीर्षक, एलएलएम विश्लेषण और पुश सामग्री के लिए एकीकृत अनुवाद पाइपलाइन (कोई भी लक्ष्य भाषा)।
- मल्टी-चैनल सूचनाएं: `फीशू`, `वीवर्क`, `वीचैट`, `टेलीग्राम`, `डिंगटॉक`, `एनटीएफवाई`, `बार्क`, `स्लैक`, `ईमेल`।
- स्मार्ट पुश रणनीतियाँ: `दैनिक` / `वृद्धिशील` / `रियलटाइम`।
- शेड्यूल किए गए फ़ेचिंग और वैकल्पिक ऑटो-पुश सदस्यता कार्य।
- प्रोटोकॉल-आधारित एजेंट एकीकरण के लिए एमसीपी टूल इंटरफ़ेस।
- प्रत्यक्ष, स्क्रिप्ट योग्य स्थानीय टूल आमंत्रण के लिए एजेंट सीएलआई (नया)।

## 🌐 एकीकृत अनुवाद पाइपलाइन (कोई भी लक्ष्य भाषा)
- सभी परतों में एक पाइपलाइन: `शीर्षक`, `एलएलएम सारांश/विश्लेषण`, और `अधिसूचना पेलोड`।
- लक्ष्य भाषा समर्थन: `अंग्रेजी`, `कोरियाई`, `जापानी`, `फ़्रेंच`, `चीनी`, `पारंपरिक चीनी`, साथ ही कस्टम भाषा लक्ष्य।
- लागत नियंत्रण: बैच अनुवाद, वृद्धिशील फ़ील्ड भरण, और बार-बार एपीआई कॉल से बचने के लिए निरंतर पुन: उपयोग।
- सुसंगत व्यवहार: वेब, एमसीपी और सीएलआई समान `आउटपुट_लैंग्वेज` शब्दार्थ साझा करते हैं, इसलिए यूआई और पुश आउटपुट सिंक में रहते हैं।

## ⚡ त्वरित शुरुआत
### 1) 🧩आवश्यकताएँ
- पायथन `>=3.12`
- `uv` की अनुशंसा की जाती है
- डॉकर उपयोगकर्ता: `डॉकर` + `डॉकर कंपोज़`

### 2) 🖥️ स्थानीय स्टार्टअप (विकास)
```bash
uv sync --locked
```

1. मुख्य फ़ेच/रनटाइम को एक बार चलाएँ:
```bash
omnihawk-ai
```

2. इंटरैक्टिव वेब कंसोल चलाएँ (6-पेज यूआई):
```bash
python -m omnihawk_ai.web.panel_server --port 8080 --output-dir output
```

3. एमसीपी सेवा (HTTP) प्रारंभ करें:
```bash
omnihawk-ai-mcp --transport http --host 0.0.0.0 --port 3333
```

### 3) 🐳 डॉकर स्टार्ट
```bash
docker compose -f docker/docker-compose.yml up -d --build
```

डिफ़ॉल्ट पोर्ट:
- वेब सेवा: `WEBSERVER_PORT` (डिफ़ॉल्ट: `8080`)
- एमसीपी समापन बिंदु: `http://127.0.0.1:3333/mcp`

रुकना:
```bash
docker compose -f docker/docker-compose.yml down
```

लॉग देखें:
```bash
docker compose -f docker/docker-compose.yml logs -f
```

## 🤖 एजेंट सीएलआई (नया)
एजेंटों/स्क्रिप्टों को एमसीपी ट्रांसपोर्ट के बिना सीधे ओमनीहॉक टूल को कॉल करने की अनुमति देने के लिए, यह रेपो `omnihawk-ai-cli` जोड़ता है।

### 🎯 डिज़ाइन लक्ष्य
- एमसीपी टूल के समान क्षमता वाली सतह (समान टूल नाम और तर्क शब्दार्थ)।
- स्वचालन-अनुकूल एकीकरण के लिए JSON इनपुट और JSON आउटपुट।
- शेल स्क्रिप्ट, सीआई पाइपलाइन और एजेंट निष्पादकों के साथ अच्छी तरह से काम करता है।

### 🧪 बेसिक कमांड
```bash
omnihawk-ai-cli tools
```

### 📌 सामान्य उदाहरण
1. सभी उपलब्ध टूल और पैरामीटर्स की सूची बनाएं:
```bash
omnihawk-ai-cli tools
```

2. इनलाइन JSON के साथ एक टूल को कॉल करें:
```bash
omnihawk-ai-cli call list_scope_items --args '{"scope":"market_finance","limit":20}'
```

3. args फ़ाइल वाले टूल को कॉल करें:
```bash
omnihawk-ai-cli call upsert_scope_subscription --args-file ./payload.json
```

4. प्रोजेक्ट रूट और आउटपुट निर्देशिका को ओवरराइड करें:
```bash
omnihawk-ai-cli --project-root . --output-dir ./output call get_project_overview
```

5. कॉम्पैक्ट JSON आउटपुट (पाइपलाइन-अनुकूल):
```bash
omnihawk-ai-cli call list_scopes --compact
```

### विंडोज़ पावरशेल उदाहरण (अनुशंसित)
1. `-args-file` (सबसे विश्वसनीय) के साथ `ConvertTo-Json` का उपयोग करें:
```powershell
$payload = @{ scope = "market_finance"; limit = 20 } | ConvertTo-Json -Compress
$payload | Set-Content -Encoding utf8 .\payload.json
omnihawk-ai-cli call list_scope_items --args-file .\payload.json --compact
```

2. यहां-स्ट्रिंग आर्ग्स फ़ाइल का उपयोग करें:
```powershell
@'
{
  "scope": "frontier",
  "max_per_source": 20,
  "source_ids": ["openai-news", "anthropic-news"]
}
'@ | Set-Content -Encoding utf8 .\payload.json

omnihawk-ai-cli call fetch_scope_items --args-file .\payload.json --compact
```

3. जिन उपकरणों को किसी तर्क की आवश्यकता नहीं है उन्हें सीधे कॉल किया जा सकता है:
```powershell
omnihawk-ai-cli call get_project_overview --compact
```

### 🧾 निकास कोड
- `0`: सफलता
- `1`: रनटाइम/टूल निष्पादन त्रुटि
- `2`: अमान्य पैरामीटर, अज्ञात टूल, या विकृत JSON

### 🛠️ सीएलआई पैरामीटर और कवरेज
सीएलआई निश्चित तर्क:

| स्तर | पैरामीटर | विवरण |
| --- | --- | --- |
| वैश्विक | `-प्रोजेक्ट-रूट` | प्रोजेक्ट रूट को ओवरराइड करें |
| वैश्विक | `-आउटपुट-डीआईआर` | आउटपुट निर्देशिका को ओवरराइड करें |
| वैश्विक | `-कॉम्पैक्ट` | कॉम्पैक्ट JSON उत्सर्जित करें |
| उपकमांड | `उपकरण` | उपलब्ध उपकरणों की सूची बनाएं |
| उपकमांड | `कॉल <टूल>` | एक उपकरण का आह्वान करें |
| `कॉल` विकल्प | `--आर्ग्स` | इनलाइन JSON तर्क |
| `कॉल` विकल्प | `--आर्ग्स-फ़ाइल` | फ़ाइल से JSON तर्क लोड करें |

टूल व्यवसाय तर्क प्रति टूल परिभाषित हैं। उपयोग:

```bash
omnihawk-ai-cli tools --compact
```

कवरेज सीमा:
- अवलोकन, स्कोप फ़ेच/लिस्ट/सेटिंग्स/सब्सक्रिप्शन, और पेपर इंटेलिजेंस/सब्सक्रिप्शन सहित वर्तमान में उजागर सभी एमसीपी टूल क्षमताओं (22 टूल्स) को कवर करता है।
- सीधे प्रक्रिया/कंटेनर जीवनचक्र का प्रबंधन नहीं करता (उदाहरण के लिए डॉकर या वेब सर्वर प्रक्रियाओं को शुरू/रोकना)।
- सीधे ब्राउज़र यूआई इंटरैक्शन नहीं करता है, लेकिन समतुल्य डेटा-लेयर ऑपरेशन (सेटिंग्स, फ़ेच, सब्सक्रिप्शन, पुश) को कवर करता है।

## ❓अक्सर पूछे जाने वाले प्रश्न
### Q1: सीएलआई और एमसीपी के बीच क्या संबंध है?
सीएलआई सीधे उसी एमसीपी-संगत टूल फ़ंक्शन को कॉल करता है। प्रोटोकॉल एकीकरण के लिए एमसीपी और स्थानीय स्क्रिप्टिंग/स्वचालन के लिए सीएलआई का उपयोग करें।

### Q2: क्या सीएलआई सभी सुविधाओं को कवर कर सकता है?
सीएलआई वर्तमान में एमसीपी (22 टूल्स) द्वारा उजागर की गई सभी क्षमताओं को कवर करता है। यदि यूआई में कोई क्रिया डेटा-लेयर ऑपरेशंस (फ़ेच, क्वेरी, सेटिंग्स, सब्सक्रिप्शन, पुश) पर मैप होती है, तो यह आमतौर पर सीएलआई के माध्यम से स्वचालित होती है।

### Q3: मुझे डुप्लिकेट संदेश क्यों दिखाई दे रहे हैं?
जाँच करना:
- क्या `आउटपुट/ai_progress_seen.json` सही ढंग से कायम/माउंट किया गया है
- क्या डुप्लिकेट सदस्यता नियम कॉन्फ़िगर किए गए हैं
- क्या `टाइमलाइन` और पेज-स्तरीय सदस्यताएँ ओवरलैपिंग विंडो को ट्रिगर कर रही हैं

### Q4: न्यूनतम एजेंट एकीकरण पथ क्या है?
के साथ शुरू:
```bash
omnihawk-ai-cli call get_project_overview
```
फिर आवश्यकतानुसार `list_scope_items` / `list_papers` / `run_*_subscriptions` को कॉल करें।
